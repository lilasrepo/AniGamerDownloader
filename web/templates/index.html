手動添加任務

一、如何取得並貼上 Cookie

下載 VIP 1080P 需要登入帳號。本程式以瀏覽器的 Cookie: 標頭代替帳密登入，步驟如下：

  1. 用電腦瀏覽器（Chrome／Edge／Firefox 皆可）登入巴哈姆特動畫瘋，確認右上角顯示為已登入的 VIP 帳號。
  2. 隨意開啟一部動畫的播放頁（網址形如 https://ani.gamer.com.tw/animeVideo.php?sn=12345）。
  3. 按 F12 開啟開發者工具，切到 Network（網路） 分頁，重新整理一次頁面。
  4. 在請求清單中點任一筆對 ani.gamer.com.tw 的請求，於 Request Headers（請求標頭） 中找到 cookie: 那一整列。
  5. 複製 cookie: 後面整串值（形如 ANIME_SIGN=...; BAHARUNE=...; ...）。
  6. 在同一個 Network 請求的標頭中，一併找到 user-agent: 並複製它的值（UA）。
  7. 到本程式的「設定」分頁，在「帳號 Cookie」區塊把 cookie 字串貼進文字框，並在上方「請求UA」欄位貼上 UA（或按「取得當前UA」自動帶入目前這個瀏覽器的 UA），按「儲存 Cookie 與 UA」。
重點
Cookie 與 UA 必須來自同一個瀏覽器。若 UA 與產生 cookie 的瀏覽器不符，巴哈會視為異常登入。
Cookie 會過期；之後下載失敗或畫質掉回低畫質時，重做上述步驟更新 cookie 即可。
Cookie 等同登入憑證，請勿外流。本程式僅將其存於本機（或掛載 volume）的 cookie.txt。

二、設定下載清單（sn_list）

「下載清單」分頁就是自動追番的來源。每行一部番劇：

  1. 從動畫播放頁網址 ?sn= 後面的數字就是該集的 sn 碼。
  2. 每行格式：sn碼 下載模式 <自訂番名> #註解，後三者皆可省略。
  3. 下載模式常用：all（整部）、latest（最後一集）、largest-sn（最近上傳）；省略則用「設定」頁的預設模式。
  4. 以 @分類名 起一行，其下的番劇會歸到該分類資料夾；單獨 @ 表示不分類。
  5. # 之後的文字程式不會讀取，可用來標記番名備註。

編輯完按「提交」即存檔。可到「監控」分頁按「立即檢查」讓程式馬上掃描一次。

三、解析度與輸出資料夾

  • 下載解析度：在「設定」分頁選擇（最高 1080P，需 VIP cookie）。開啟「鎖定解析度」則找不到該畫質時不自動降級。
  • 下載目錄：影片最終存放處。留空則存在程式所在資料夾下的 data/bangumi。
  • 暫存目錄：下載與合併過程的暫存區。建議與下載目錄放在同一個磁碟，完成後搬移才會是瞬間（跨磁碟會慢）。留空則回退到 data/temp。
  • 「建立番劇資料夾」開啟時，每部番劇各自一個資料夾，較好整理。

四、Docker 版（NAS／mac／Linux）

Docker 版以背景 daemon + 本網頁介面運作，設定與輸出皆透過掛載 volume：

  • 掛載 /data：存放 config.json、cookie.txt、資料庫與日誌。第一次啟動會自動產生預設 config.json。
  • 掛載一個輸出目錄（對應 NAS 上的下載資料夾），並在「設定」頁把「下載目錄」指到容器內對應的掛載路徑。
  • 容器內無法手動編輯 cookie.txt，請改用本介面「設定」分頁的「帳號 Cookie」區塊貼上。
  • 密碼保護（BasicAuth）：Docker 版強制啟用簡易密碼，由環境變數設定；空密碼會被拒絕啟動。請只在區域網路內使用，勿對外網直接暴露。
  • 首次連上後，依序：貼 cookie → 設下載目錄 → 編輯下載清單 → 「立即檢查」開始追番。

五、常見問題

下載失敗或畫質掉回低畫質
多半是 cookie 過期或與 UA 不符。重新取得 cookie 與 UA（同一瀏覽器）後再儲存。
合併／轉檔失敗
本程式需要 ffmpeg。桌面版請確認 ffmpeg 在 PATH 或放在程式旁；Docker 映像已內含。
檔名出現 E01 與舊的 E1 混在一起
集數補零規則已改為「預設兩碼、破百自動三碼」。舊的下載檔仍是舊命名，資料庫盤點時可能把舊檔當成缺漏；必要時手動改名或忽略。
設定改了沒生效
記得按「儲存」。部分設定會在下一輪檢查才套用，可到「監控」按「立即檢查」加速。
反向代理掛在子路徑無法運作
本介面部分請求使用根路徑，建議 Docker 版掛在反向代理的根路徑，不要掛在子目錄。
